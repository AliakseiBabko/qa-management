# Template: Test Strategy

Output format — Google Doc, one living document per project under
`30_Project_Knowledge\<Project>\qa_docs\`, named `test_strategy`. A
downstream knowledge product, not an automatic output of every source -
update only when a project-knowledge source clearly affects overall test
approach.

## <Project> — Test Strategy

### Overall Approach

The project's testing philosophy - manual vs. automated balance, risk-based
prioritization, etc.

### Tooling

Frameworks/tools in use or planned.

### Automation Approach

What's automated, what isn't, and why.

### Reporting/Metrics

How test results and quality signals get communicated.

### Risks

Known risks to the overall test approach.

### Open Questions

Anything still unresolved about strategy.
