# Template: Test Plan

Output format — Google Doc, one living document per project under
`30_Project_Knowledge\<Project>\qa_docs\`, named `test_plan`. A downstream
knowledge product, not an automatic output of every source - update only
when a project-knowledge source clearly affects test scope.

## <Project> — Test Plan

### Scope

What's in and out of scope for testing.

### Test Levels

Unit / integration / system / acceptance, as applicable to this project.

### Environment/Data

Test environment(s) and data requirements.

### Entry/Exit Criteria

When testing can start, and when it's considered done.

### Risks

Known risks to test coverage or execution.

### Open Questions

Anything still unresolved about scope or approach.
