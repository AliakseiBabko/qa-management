# Шаблон: Входные данные M2 для сведения проекта в rollup

Формат вывода — Google Doc. Один живой документ на проект — не создавай
новый файл на каждый цикл. Раунды добавляются по датам сверху вниз (новый
раунд — новая секция), старые раунды не удаляются и не архивируются:
история видна прямо в документе, а Google Docs version history — это
дополнительная подстраховка, не замена видимой истории.

## <Проект> — входные данные M2

Каждый раунд соответствует одному циклу сведения `project_development_plan`/
`project_risk` из индивидуальных планов, метрик и `Вклад в проект`.

## Раунд: <дата>

### Вопросы от предварительного анализа

Заполняется M2-агентом до того, как M2-человек написал что-либо в этом
раунде. Это результат просмотра индивидуальных планов/метрик всех людей на
проекте перед сведением их в project-level rollup: что неясно, каких данных
не хватает, где конкретно нужен человеческий контекст, которого нет ни в
одной метрике (например: расхождение между тем, что показывают метрики
конкретного человека, и общим ощущением по проекту; риск, который не виден
ни в одной метрике; что делать с человеком, для которого core-метрики ещё
не собираются). Вопросы должны быть конкретными и такими, на которые можно
ответить одним-двумя предложениями — не общими «как дела на проекте».

### Ответ и общие соображения M2

Заполняется M2-человеком: общее мнение о ситуации на проекте (что важно,
что нет — не обязано быть привязано к вопросам выше) и прямые ответы на
вопросы из раздела выше. Если раздел пуст — rollup для этого раунда ещё не
может быть выполнен, нужно дождаться ответа.
