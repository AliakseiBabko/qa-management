# Template: Performance Test Plan

Output format — Google Doc, one living document per project under
`30_Project_Knowledge\<Project>\qa_docs\`, named `performance_test_plan`.
A downstream knowledge product, not an automatic output of every source -
update only when a project-knowledge source clearly affects performance
testing scope/approach.

## <Project> — Performance Test Plan

### Scope

Which flows/components are in scope for performance testing, and why
(see the knowledge base's Performance-Critical Scenarios section).

### Test Types

Load / stress / soak / spike / other, as applicable.

### Environment/Tooling

Target environment(s) and tooling planned or in use.

### Success Criteria

What "acceptable performance" means for this project, concretely.

### Risks

Known risks to executing this plan (data availability, environment
access, tooling gaps).

### Schedule

Planned timing, if known.

### Open Questions

Anything still unresolved before this plan can be executed.
