# QA Management Roles

Use this skill as shared context. It does not own a final report format.

## Required Start

1. Identify whether the task is M1 people management, M2 project management, or mixed.
2. Read the relevant reference:
   - `references/m1-role-rules.md`
   - `references/m2-role-rules.md`
3. Apply the role boundary before writing outputs or conclusions.

## Role Levels

- M1: people management, approximately 3-8 people.
- M2: project management, approximately 3-5 projects and 5-10 people.
- M3: subdivision lead, approximately 4-6 M1 and 2-4 M2.
- M4: Head, responsible for strategic planning and effectiveness of the full QA department.

## Guardrails

- Do not mix people-management conclusions with project-management conclusions without labeling the role perspective.
- Do not treat QA task execution as M2 value by itself. M2 value must connect to project, client, business, staffing, role growth, or measurable delivery outcomes.
- Do not invent missing evidence. Ask for it or leave fields blank when the output schema allows blanks.
