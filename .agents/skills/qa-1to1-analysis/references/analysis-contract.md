# Контракт анализа

## Назначение

Использовать этот файл как общий контракт структурированного анализа QA 1to1-транскрипта.

## Минимальный набор выводов

- дата встречи, если она известна
- сотрудник или сотрудники, о которых идет речь
- доминирующая тема встречи
- самые сильные факты из транскрипта
- релевантные сигналы для people management
- релевантные сигналы для project management
- конкретные follow-up действия

## Правило доказательности

- Предпочитать прямые факты и примеры из транскрипта.
- Если транскрипт расплывчатый, писать это прямо, а не заполнять выводы абстракциями.
- Делать анализ переиспользуемым для downstream skills.

## Язык

- По умолчанию писать выводы на русском языке.
- Английские термины, определения и цитаты можно сохранять, если они есть в источнике или являются нормальной рабочей терминологией.
- Если пользователь явно просит другой язык, следовать запросу пользователя.

## Использование downstream skills

- M1 и M2 1to1 writer skills могут использовать одну и ту же схему `Templates/1to1.csv`.
- Сохранять анализ нейтральным к папке назначения. Не выбирать output folder в этом skill.
- Извлекать и people-management, и project-management сигналы, если они есть, даже если текущий запрос пишет только один итоговый файл.
